Characters Endpoints

Endpoint Method Accessible By JSON Description
/characters GET Admin, User N/A

Postman: getAllCharacters


This endpoint allows you to view all Marvel characters currently in de database.
/characters/{characterId} GET Admin, User N/A

Postman: getCharacterById


This endpoint allows you to view a specific Marvel character by providing its id in the location of the {characterId} path variable.

Example: /characters/1000
/characters/search/starting-with GET Admin, User N/A

Postman: findCharactersByNameStartingWith


This endpoint allows you to search for characters with a query parameter. This method searches for characters starting their name as filled in as query parameter.

Example: /characters/search/starting-with?name=vision
/characters/search/contains GET Admin, User N/A

Postman: findCharactersByNameContains


This endpoint allows you to search for characters with a query parameter. This method searches for characters whose name contains that which was filled in as query parameter.

Example: /characters/search/contains?name=vision
/characters/search/sorted-asc GET Admin, User N/A

Postman: findCharactersByNameSortedAsc


This endpoint allows you to search for characters with a query parameter. This method returns the search results in ascending order.

Example: /characters/search/sorted-asc?name=vision
/characters/search/sorted-desc GET Admin, User N/A

Postman: findCharactersByNameSortedDesc


This endpoint allows you to search for characters with a query parameter. This method returns the search results in descending order.

Example: /characters/search/sorted-desc?name=vision
/characters/search/actor-name GET Admin, User N/A

Postman: findCharactersByActorNameContains


This endpoint allows you to search for characters with a given query parameter. This method searches for characters that are played by certain actors and whose name contains that which was filled in as query parameter.

Example: /characters/search/actor-name?name=robert
/characters/character/{characterId} PUT Admin {
"characterAliasName": "Hulk",
"characterRealName": "Robert Bruce Bannerz",
"characterActorName": "Mark Ruffalo",
"characterTitle": "The incredible Hulk",
"characterGender": "Male",
"characterSummary": "Exposed to heavy doses of gamma radiation, scientist Bruce Banner transforms into the mean, green rage machine called the Hulkz.",
"characterDescription": "Dr. Bruce Banner lives a life caught between the soft-spoken scientist he’s always been and the uncontrollable green monster powered by his rage."
}

Postman: updateCharacterById


Ths endpoint allows you to update a character. The character id should be entered in the location of the {characterId} path variable. Enter the required character information into the JSON body.

The gender value has 3 possible options: Male, Female and X.

Example: /characters/character/1000
/characters/associate/movie/{characterId} PUT Admin {
"id": 1000
}

Postman: associateMovieAndCharacter


This endpoint allows you to combine a character and a movie. The character will then be added to that movie and that movie will be added to that character. The id in the JSON body is the id of the movie. The character id should be entered in the location of the {characterId} path variable.

Example: /characters/associate/movie/1007
/characters POST Admin {
"characterAliasName": "She-Hulk",
"characterRealName": "Jennifer Walters",
"characterActorName": "Tatiana Maslany",
"characterTitle": "Savage She-Hulk",
"characterGender": "Female",
"characterSummary": "Jennifer Walters is a talented lawyer and the cousin of Bruce Banner, the Hulk",
"characterDescription": " After being shot by a mobster and seriously injured, she was saved by a blood transfusion from Bruce, and his gamma-irradiated blood mutated Jennifer into the She-Hulk, kicking off her adventuring career."
}

Postman: createCharacter


This endpoint allows you to create a character with the specified values and attributes.

The gender value has 3 possible options: Male, Female and X.
/characters/{characterId} DELETE Admin N/A

Postman: deleteCharacterById


This endpoint allows you to delete a character with a certain id. The character id should be entered in the location of the {characterId} path variable.

Note: Currently, characters assigned to users (Favorites) or movies, are not allowed to be deleted.

Use the removeFavoriteCharacterFromUser and disassociateMovieAndCharacter methods before trying to remove the character.

Example: /characters/1000
/characters/disassociate/movie/{characterId} DELETE Admin {
"id": 1005
}

Postman: disassociateMovieAndCharacter


This endpoint allows you to disassociate a character and a movie. The character will then be removed from that movie and that movie will be removed from that character. The id in the JSON body is the id of the movie. The character id should be entered in the location of the {characterId} path variable.

Example: /characters/disassociate/movie/1000

/users/admin/{username} PUT Admin {
"password": "bAn@nen1"
}

Postman: updateUserPasswordAdmin (Admin only)


This endpoint allows the admin to overrule a user's password by providing a new password in the JSON Body. The username of the user should be entered in the location of the {username} path variable.

Password needs to contain the following:
1. Minimum of 1 lowercase letter.
2. Minimum of 1 uppercase letter.
3. Minimum of 1 number.
4. Minimum of 1 symbol.
5. It should be between 8 and 20 characters long.

Example: /users/admin/andre
/users/user/{username} PUT User {
"password": "bAn@nen2",
"apikey": "12345678",
"email": "andre@novi.nl"
}

Postman: updateUserDetails (User only)


This endpoint allows the user to update his or her account. Update its user details, including password, by providing the required details in the JSON body. The username of the user should be entered in the location of the {username} path variable.

Password needs to contain the following:
1. Minimum of 1 lowercase letter.
2. Minimum of 1 uppercase letter.
3. Minimum of 1 number.
4. Minimum of 1 symbol.
5. It should be between 8 and 20 characters long.

Example: /users/user/andre
/users/assign/profile/{username} PUT Admin {
"id": 1
}

Postman: assignProfileToUser

/users/{username} DELETE Admin, User N/A

Postman: deleteUser


This endpoint allows the admin (or user) to delete a user by entering the desired username. The username of the user should be entered in the location of the {username} path variable.

Currently, users that have profiles or contact forms assigned to them are not allowed to be deleted. First use the detachProfileFromUser method or try the deleteContactFormById method and then try again to delete the user.

Example: /users/andre
/users/{username}/authorities/{authority} DELETE Admin N/A

Postman: removeUserAuthority


This endpoint allows the admin to demote an admin to user by entering the desired username and ROLE_ADMIN. The username and authority (ROLE_ADMIN) of the user should be entered in the locations of the {username} and {authority} path variables.

Example: /users/andre4/authorities/ROLE_ADMIN
/users/remove/character/{username} DELETE Admin, User N/A

Postman: removeFavoriteCharacterFromUser


This endpoint allows the admin or user to remove a Marvel character from his or her favorites list. The username of the user should be entered in the location of the {username} path variable.

Example: /users/remove/character/andre
