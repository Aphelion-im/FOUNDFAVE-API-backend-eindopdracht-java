Characters Endpoints

Endpoint Method Accessible By JSON Description
/characters GET Admin, User N/A

Postman: getAllCharacters


This endpoint allows you to view all Marvel characters currently in de database.
/characters/{characterId} GET Admin, User N/A

Postman: getCharacterById


This endpoint allows you to view a specific Marvel character with an id.
/characters/search/starting-with GET Admin, User N/A

Postman: findCharactersByNameStartingWith


This endpoint allows you to search for characters with a query parameter. This method searches for characters starting their name as filled in as query parameter.
/characters/search/contains GET Admin, User N/A

Postman: findCharactersByNameContains


This endpoint allows you to search for characters with a query parameter. This method searches for characters whose name contains that which was filled in as query parameter.
/characters/search/sorted-asc GET Admin, User N/A

Postman: findCharactersByNameSortedAsc


This endpoint allows you to search for characters with a query parameter. This method returns the search results in ascending order.
/characters/search/sorted-desc GET Admin, User N/A

Postman: findCharactersByNameSortedAsc


This endpoint allows you to search for characters with a query parameter. This method returns the search results in descending order.
/characters/search/actor-name GET Admin, User N/A

Postman: findCharactersByActorNameContains


This endpoint allows you to search for characters with a query parameter. This method searches for characters that are played by certain actors and whose name contains that which was filled in as query parameter.
/characters/character/{characterId} PUT Admin {
"characterAliasName": "Hulk",
"characterRealName": "Robert Bruce Bannerz",
"characterActorName": "Mark Ruffalo",
"characterTitle": "The incredible Hulk",
"characterGender": "Male",
"characterSummary": "Exposed to heavy doses of gamma radiation, scientist Bruce Banner transforms into the mean, green rage machine called the Hulkz.",
"characterDescription": "Dr. Bruce Banner lives a life caught between the soft-spoken scientist he’s always been and the uncontrollable green monster powered by his rage."
}

Postman: updateCharacterById


Ths endpoint allows you to update a character with a specific id and providing the required information to the JSON body. The gender value has 3 possible options: Male, Female and X.
/characters/associate/movie/{characterId} PUT Admin {
"id": 1000
}

Postman: associateMovieAndCharacter


This endpoint allows you to combine a character and a movie. The character will then be added to that movie and that movie will be added to that character. The id in the JSON body is the id of the movie.
/characters POST Admin {
"characterAliasName": "She-Hulk",
"characterRealName": "Jennifer Walters",
"characterActorName": "Tatiana Maslany",
"characterTitle": "Savage She-Hulk",
"characterGender": "Female",
"characterSummary": "Jennifer Walters is a talented lawyer and the cousin of Bruce Banner, the Hulk",
"characterDescription": " After being shot by a mobster and seriously injured, she was saved by a blood transfusion from Bruce, and his gamma-irradiated blood mutated Jennifer into the She-Hulk, kicking off her adventuring career."
}

Postman: createCharacter


This endpoint allows you to create a character with the specified values and attributes. The gender value has 3 possible options: Male, Female and X.
/characters/{characterId} DELETE Admin N/A

Postman: deleteCharacterById


This endpoint allows you to delete a character with a certain id. Note: Currently, characters assigned to users (Favorites) or movies, are not allowed to be deleted. Use the removeFavoriteCharacterFromUser and disassociateMovieAndCharacter methods before trying to remove the character.
/characters/disassociate/movie/{characterId} DELETE Admin {
"id": 1005
}

Postman: disassociateMovieAndCharacter


This endpoint allows you to disassociate a character and a movie. The character will then be removed from that movie and that movie will be removed from that character. The id in the JSON body is the id of the movie.

Postman: getUserAuthorities


This endpoint allows you to see which roles (Admin or User) a user has by providing the username.
/users/admin/{username} PUT Admin {
"password": "bAn@nen1"
}

Postman: updateUserPasswordAdmin (Admin only)


This endpoint allows you to overrule a user's password by providing a new password in the JSON Body.

Password needs to contain the following:
1. Minimum of 1 lowercase letter.
2. Minimum of 1 uppercase letter.
3. Minimum of 1 number.
4. Minimum of 1 symbol.
5. It should be between 8 and 20 characters long.
/users/user/{username} PUT User {
"password": "bAn@nen2",
"apikey": "12345678",
"email": "andre@novi.nl"
}

Postman: updateUserDetails (User only)


This endpoint allows the user to update his or her user details, including password, by providing the required details in the JSON body.

Password needs to contain the following:
1. Minimum of 1 lowercase letter.
2. Minimum of 1 uppercase letter.
3. Minimum of 1 number.
4. Minimum of 1 symbol.
5. It should be between 8 and 20 characters long.
/users/assign/profile/{username} PUT Admin {
"id": 1
}

Postman: assignProfileToUser


This endpoint allows the admin to assign a new profile to a user by providing the required profile id in the JSON body.
/users/detach/profile/{username} PUT Admin N/A

Postman: detachProfileFromToUser


This endpoint allows the admin to detach a profile from a user's account. Only detached profiles can be deleted.
